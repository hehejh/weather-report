## 1. 项目概述

### 1.1 产品愿景

打造摄影爱好者首选的天气决策工具——让每一次拍摄都有好天气。

### 1.2 目标用户画像

| 维度 | 描述 |
|------|------|
| **核心用户** | 风光摄影爱好者、星空摄影师、城市风光摄影师 |
| **年龄** | 20-45 岁 |
| **设备** | 智能手机为主，部分用户希望有桌面端 |
| **行为特征** | 提前规划拍摄行程，关注天气变化，愿意为出片付出时间成本 |
| **痛点** | 通用天气 App 不提供朝霞晚霞、云量分层等摄影关键信息，摄影规划 App 缺少天气集成 |

### 1.3 核心价值主张

- **一则看尽摄影天气** — 温度、降水、云量、能见度、霞光概率、日出日落、月相一站聚合
- **地图即画布** — 在地图上收藏摄影点位，关联天气信息，建立个人的摄影点位地图
- **不错过好天气** — 当收藏点位可能出现好摄影条件时，提前一天智能提醒

### 1.4 差异化定位

| 竞品 | 我们的优势 |
|------|-----------|
| 莉景天气 | 地图收藏点位 + 天气关联，不只是看天气而是管理摄影计划 |
| PhotoPills | 内置天气数据，无需切换 App；中国区数据更准确 |
| 通用天气 App | 摄影专属纬度（霞光、云量分层、通透度） |

### 2.2 竞品详细分析

#### 莉景天气
- **优势:** 23项景观预报行业最全，分层云量独一无二，ECMWF数据源质量高，中国3000+城市覆盖
- **劣势:** 无地图收藏点位功能，UI设计偏传统，无社交属性
- **启示:** 景观预报的广度是我们的参考基准

#### PhotoPills
- **优势:** 摄影规划黄金标准，AR天体追踪无可替代，丰富的计算器工具，一次性买断
- **劣势:** 无天气数据集成（用户强烈期望），学习曲线陡峭，无桌面版
- **启示:** 我们需要做的是将天气数据带入摄影规划工作流

#### 巧摄 PlanIt
- **优势:** 中文摄影规划标杆，功能全面（景深、全景、星轨计算等），价格便宜
- **劣势:** 天气功能较弱，UI 复杂度高
- **启示:** 中文市场存在摄影规划+天气一体化的明确空白

#### Alpenglow / SkyCandy
- **优势:** SunsetWx 驱动的朝霞晚霞质量预测，简洁美观
- **劣势:** 仅 iOS，仅霞光预报（无综合天气），提醒功能需订阅
- **启示:** 霞光预报有成熟 API，体验可以做得很好

### 2.3 市场机会

- **中国风光摄影爱好者保守估算:** 500 万+（基于摄影社区 MAU 推算）
- **现有缺口:** 没有一款应用同时做好"地图收藏点位"+"摄影天气"+"智能提醒"三者集成
- **差异化窗口:** 莉景做好了天气但没做点位管理，PhotoPills 做好了点位管理但没有天气，我们在中间位置

## 3. 用户需求分析

### 3.1 用户旅程

```
规划出行  →  收藏点位  →  等待时机  →  收到提醒  →  出发拍摄
    │              │              │              │              │
    ▼              ▼              ▼              ▼              ▼
选择拍摄     在地图上标记    定期查看天气    ※前一日收到   到达点位
目的地/主题   理想拍摄位置    判断是否适合    智能推送通知   确认天气
                             拍摄            确认出行       完成拍摄
```

### 3.2 按摄影类型细分需求

| 摄影类型 | 核心天气关注点 | 关键阈值 |
|---------|--------------|---------|
| **风光摄影** | 云量(高云有利)、朝霞晚霞概率、能见度、风速 | 高云量 30-70%，风速 <5级 |
| **星空摄影** | 云量(越低越好)、月相、视宁度、光污染 | 低云量 <20%，月相避开满月前后 |
| **城市风光** | 能见度、空气质量(AQI)、晚霞概率 | AQI <100，能见度 >5km |
| **人像摄影** | 光比(柔光条件)、降水概率、温度 | 黄金时段、无雨、温度适宜 |
| **海景摄影** | 潮汐、风速、浪高、朝霞晚霞 | 配合潮汐+霞光时间 |

### 3.3 提醒时机偏好

| 提醒类型 | 时机 | 原因 |
|---------|------|------|
| 朝霞提醒 | 前一日 20:00-21:00 | 用户需要提前准备早起 |
| 晚霞提醒 | 当日 12:00-14:00 | 当天中午能决定下午出发 |
| 星空提醒 | 前一日 18:00 | 夜间出行需提前安排 |
| 极端天气提醒 | 提前 24h | 安全考虑 |

### 3.4 用户关键洞察

基于同类产品用户评价和摄影社区讨论：

1. **"出门前看一眼就能决定带不带相机"** — 信息呈现要快，避免过载
2. **"我希望在地图上看到我所有踩过点的地方，点一下就知道明天适不适合去"** — 地图是信息组织的核心
3. **"霞光预报不太准，但总比没有好"** — 提醒需要标注置信度
4. **"冬天和夏天的好天气标准不一样"** — 季节因素影响判断阈值
5. **"有时候想拍雾，有时候想拍通透，天气的好坏取决于我想拍什么"** — 需要按拍摄意图自定义条件
6. **"免费的最多用一天几百次查询，但我不介意付费，只要好用"** — 存在明确付费意愿

## 4. 功能需求

### 4.1 功能优先级总览

| 优先级 | 定义 | 包含功能 |
|-------|------|---------|
| **P0 MVP** | 必须实现，否则产品不可上线 | 地图选点、基础天气、霞光预报、简单提醒 |
| **P1 V1.0** | 核心体验完善后上线 | 提醒自定义、月相、星空指数、点位详情增强 |
| **P2 V2.0** | 增强体验和商业化 | 社交分享、社区点位、API 开放 |

### 4.2 P0 — MVP 功能

#### F-01: 地图收藏摄影点位

| 维度 | 说明 |
|------|------|
| **描述** | 在地图上长按或搜索地址来标记摄影点位，赋予名称和标签 |
| **交互** | 地图长按 → 弹出标记面板 → 输入名称、选标签 → 保存 |
| **标签预设** | 风光、星空、城市、人像、海景、日出、日落、自定义 |
| **验收条件** | 1) 地图上可看到所有已收藏点位 2) 点击点位弹出基本信息卡片 3) 支持滑动切换点位 |

#### F-02: 摄影天气仪表盘

| 维度 | 说明 |
|------|------|
| **描述** | 针对选中点位展示多维度摄影天气信息 |
| **数据维度** | 温度、降水概率、风速/风向、云量(总+高/中/低分层)、能见度、湿度、AQI |
| **摄影要素** | 朝霞概率+质量评分、晚霞概率+质量评分、日出/日落时间、黄金时刻/蓝调时刻 |
| **呈现方式** | 卡片式仪表盘，关键指标颜色编码（绿=适合、黄=一般、红=不适合） |
| **验收条件** | 1) 选点后 3 秒内加载天气 2) 至少展示 8 项摄影指标 3) 未来 3 天逐小时预报 |

#### F-03: 智能提醒

| 维度 | 说明 |
|------|------|
| **描述** | 当收藏点位可能出现好摄影条件时，提前一天推送通知 |
| **触发条件(MVP)** | 朝霞概率 >60% 或 晚霞概率 >60% |
| **推送时机** | 朝霞：前一日 20:00，晚霞：当日 12:00 |
| **推送内容** | "📍 [点位名称] 明日可能有高质量朝霞（概率 78%），日出 05:23，建议 04:30 到达" |
| **验收条件** | 1) 系统每日自动检查所有收藏点位 2) 满足条件时推送 3) 点击通知跳转天气详情 |

#### F-04: 首页概览

| 维度 | 说明 |
|------|------|
| **描述** | 打开应用即看到关键信息：地图+收藏点位+今日/明日摄影指数 |
| **布局** | 顶部显示当前位置摄影指数评分，地图占据主体，底部展示最近点位天气摘要 |
| **验收条件** | 1) 冷启动 2 秒内完成 2) 默认定位当前位置 3) 摄影指数每日更新 |

### 4.3 P1 — V1.0 功能

- **F-05: 提醒自定义规则** — 用户可自定义触发条件组合（概率/云量/风速/温度阈值），单点位独立配置，免打扰时段
- **F-06: 星空摄影支持** — 月相日历 + 月光污染评估，银河可见时段和方位，视宁度预报
- **F-07: 点位详情增强** — 拍摄方向标记（在地图上画视野方向），参考照片上传，拍摄笔记，历史天气回顾
- **F-08: 拍摄日历** — 日历视图标记未来 7-14 天最佳拍摄日，按摄影类型过滤，点击日期查看详细天气

### 4.4 P2 — V2.0 功能

- **F-09: 社区点位分享** — 用户可公开分享点位和作品
- **F-10: 作品墙** — 关联点位+天气数据+最终成片展示
- **F-11: 团体出行协调** — 创建拍摄计划并邀请好友
- **F-12: 数据 API 开放** — 对高级用户提供原始数据导出

## 5. 非功能需求

### 5.1 性能指标

| 指标 | 目标 |
|------|------|
| 冷启动时间 | < 2 秒 |
| 天气数据加载 | 缓存 <200ms，API 查询 <3 秒 |
| 地图渲染 | 100 点位以内 <1 秒 |
| 提醒扫描定时任务 | 全量用户扫描 <5 分钟 |

### 5.2 数据更新频率

| 数据类型 | 更新频率 | 来源 |
|---------|---------|------|
| 基础天气 | 每小时 | 和风天气 / OpenWeatherMap |
| 朝霞晚霞概率 | 每 6 小时 | SunsetWx / 彩云天气(企业) |
| 日出日落时间 | 每日计算 | 天文算法（无需 API） |
| 空气质量 | 每小时 | 和风天气 |
| 提醒扫描 | 每日 06:00、18:00 | 定时任务 |

### 5.3 可靠性

- 提醒推送到达率 > 95%
- 天气数据不可用时使用缓存 + 标注"数据可能过期"
- API 调用失败自动重试 3 次，间隔 1 分钟

### 5.4 安全性

- 用户位置数据加密存储
- 收藏点位默认私有，用户主动选择是否公开
- API Key 后端代理，不暴露给客户端
- 遵守《个人信息保护法》，位置数据不用于其他目的

## 7. 天气数据源与技术可行性

### 7.1 天气 API 选型评估

| API | 免费额度 | 摄影核心数据 | 朝霞/晚霞 | 付费价格 | 覆盖 | 推荐度 |
|-----|---------|------------|----------|---------|------|-------|
| **和风天气** | 5万次/月 | 云量、能见度、AQI、日出日落、湿度 | **无直接API** | 0.0007元/次起 | 全球 | ⭐⭐⭐⭐⭐ |
| **彩云天气** | 1000次/天 | 云量、能见度、CAPE | **有但仅企业** | 2498元/月起 | 全球(霞光仅中国) | ⭐⭐⭐⭐ |
| **OpenWeatherMap** | 1000次/天 | 云量、能见度、UV、露点 | 无 | 免费层起步 | 全球 | ⭐⭐⭐⭐ |
| **SunsetWx** | 免费注册 | **霞光质量评分** | **专有** | 联系获取 | 全球 | ⭐⭐⭐⭐⭐ |
| **Visual Crossing** | 免费层 | 太阳辐射、云量、能见度 | 无 | 按量付费 | 全球 | ⭐⭐⭐ |

### 7.2 推荐数据源组合

**MVP 阶段（低成本启动）:**

```
和风天气 (基础天气 + AQI) + SunsetWx (霞光质量评分) = 核心数据全覆盖
```

- 和风天气 5 万次/月免费额度足以支撑 MVP 阶段数千 DAU
- SunsetWx 免费注册即可使用，直接返回霞光质量评分（Poor/Fair/Good/Great）
- 日出日落/黄金时刻可通过天文算法本地计算，无需 API 调用

**缩放方案 (V1.0+):**

```
主数据源: 和风天气 (付费) + 彩云天气 (企业套餐)
备数据源: OpenWeatherMap (降级备份)
霞光数据: SunsetWx + 自研算法 (混合模式)
```

### 7.3 霞光预报 — 自研算法预案

当 SunsetWx 不可用或彩云企业套餐成本过高时，可采用经验模型作为降级方案。

**霞光形成关键气象因子：**

1. **中高云层** (3000-8000m) — 最有利用于霞光，云量 30-70% 最佳
2. **低云** (<2000m) — 不利，会遮挡阳光
3. **相对湿度** — 40-70% 有利于色彩散射
4. **空气质量** — AQI < 80 时颗粒物散射使色彩更鲜艳
5. **风速** — 高层风速适中时云层形态更好

**简易评分模型：**

```
霞光指数 = w1×高云量得分 + w2×(1-低云量/100) + w3×湿度得分 + w4×(1-AQI/500) + w5×季节因子

- 高云量得分: 30-70% 区间满分，两端线性递减
- 湿度得分: 40-70% 区间满分，两端线性递减
- 季节因子: 秋季 > 春季 > 冬季 > 夏季
- 权重: w1=0.35, w2=0.25, w3=0.20, w4=0.10, w5=0.10
```

数据来源全部来自和风天气（云量分层、AQI、湿度），无额外 API 成本。

> **注意:** 此算法为理论模型，上线前需用 SunsetWx 或彩云天气历史数据进行校准验证。

### 7.4 技术可行性总结

| 风险项 | 等级 | 缓解措施 |
|-------|------|---------|
| 霞光数据不可用 | 中 | SunsetWx + 自研算法双轨道互为备份 |
| API 调用量超免费额度 | 低 | MVP 阶段 DAU 有限，免费额度充裕 |
| 天气预报准确率不佳 | 中 | 展示置信度区间，支持用户反馈修正 |
| 位置隐私合规 | 低 | 位置数据仅用于天气查询，后端不持久化原始坐标 |
| 跨平台开发成本 | 中 | 优先 Web App (PWA) 快速验证产品假设，再评估 Native |

## 8. 版本规划

### MVP (6-8 周)

**目标:** 验证核心价值假设——摄影爱好者是否愿意使用"地图收藏点位+天气"的组合工具。

```
Week 1-2: 项目搭建 + 地图集成 + 点位收藏 CRUD
Week 3-4: 天气 API 对接 + 摄影天气仪表盘
Week 5-6: 智能提醒逻辑 + 推送通知接入
Week 7:   集成测试 + 内测用户反馈
Week 8:   问题修复 + MVP 发布
```

**MVP 发布标准:**
- 用户可在地图上收藏/管理至少 50 个点位
- 每个点位展示不少于 8 项摄影天气指标
- 朝霞/晚霞提醒在满足条件时准确触发
- 用户可在 3 步内完成"选点→看天气→设提醒"

### V1.0 (MVP + 8 周)

- 提醒自定义规则（多条件组合、单点位独立配置）
- 星空摄影支持（月相、视宁度、银河时段）
- 拍摄日历
- 点位详情增强（照片上传、笔记、拍摄方向标记）
- 中国区全面覆盖

### V2.0 (V1.0 + 12 周)

- 社区点位分享
- 作品墙（点位+天气+成片）
- 桌面 Web 版
- 数据 API 开放
- 国际化扩展（日韩、东南亚）
